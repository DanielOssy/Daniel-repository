# Ebhohimen Daniel Osazuwa — Portfolio

A static site: no build step, no dependencies to install. Just HTML + one CSS file.

## Files
- `index.html` — main page (directory, service record, competencies, education)
- `case-file-01.html`, `case-file-02.html`, `case-file-03.html` — sample work documents
- `style.css` — shared styling

## How to put this on GitHub Pages

1. Create a new repository on GitHub (e.g. `daniel-portfolio`). Keep it **public** — Pages needs that on a free account.
2. Upload all the files in this folder to the repository (drag-and-drop works fine via the GitHub website: "Add file" → "Upload files").
3. In the repository, go to **Settings → Pages**.
4. Under "Build and deployment," set **Source** to "Deploy from a branch," branch `main`, folder `/ (root)`. Save.
5. Wait a minute, then refresh that page — GitHub will show the live URL, something like:
   `https://<username>.github.io/daniel-portfolio/`
6. Put that link on the CV and job applications.

## Editing later
Open any `.html` file in a text editor (even Notepad works) — the content is plain readable text between tags. To add a fourth case file, copy `case-file-03.html`, rename it, edit the text, and add a new row to the directory table in `index.html`.

## Note on the case files
The three sample documents are illustrative — written to demonstrate research, analysis, and documentation method, not reproductions of real confidential employer material. That's stated on the site itself (footer of the homepage and on each document) so it reads as honest rather than embellished.
